## P0 — Critical Issues (Fix immediately)

### P0-1: `any` Type Proliferation (65 instances)
**Files affected:** Throughout `src/` (54 instances) and `server/` (11 instances)
**Impact:** Bypasses TypeScript's type safety, leading to runtime errors that are invisible at compile time.
**Key offenders:**
- `server/lib/agent/claude/tool-registry.ts:179` — `SdkMcpToolDefinition<any>` for studio tools
- `server/api/chat/schema.ts` — Chat schema types untyped
- Multiple `src/services/` files

**Action:** Replace `any` with specific interfaces or use `unknown` with type narrowing.

---

## P1 — High Issues

### P1-1: Missing Auth Middleware on `/api/db/users`
**File:** `server/routes/db-users.ts:20`
**Issue:** The `GET /api/db/users` endpoint validates the query schema with Zod but has **no `requireAuth` or `requireOrgAccess` middleware**. An unauthenticated caller can query user records by email or ID.
**Action:** Add `requireAuth` before the route handler.

### P1-2: API Response Envelope Inconsistency
**Files:** Multiple route files
**Issue:** No standard `{data, error, meta}` envelope. Mixed patterns observed:
- `res.json({ schedules })` — `db-tournaments.ts`
- `res.json(jobs[0])` — `generation-jobs.ts`
- `res.json({ suggestions: ..., cached: true })` — `admin.ts`

**Action:** Adopt and enforce `{data, error, meta}` envelope via a shared `sendSuccess()` / `sendError()` helper across all routes.

### P1-3: Missing Zod Validation on Some POST Routes
**Files:** `server/routes/generation-jobs.ts`, `server/routes/health.ts`
**Issue:** `POST /api/generate/queue` and `POST /api/db/stats/reset` lack `validateRequest()` middleware for request body validation.
**Action:** Add `validateRequest()` with appropriate Zod schema to all state-changing routes.

**Files:**

- `src/components/playground/BottomPromptBar.tsx`
- `src/components/brand/BrandProfileSetup.tsx`

**Issue:** Pattern `{count && <Component/>}` renders `0` to DOM when `count === 0`.
**Action:** Replace with `{count > 0 && <Component/>}` or `{count ? <Component/> : null}`.

### P2-3: Weak HTML Sanitization in AssistantPanel
**File:** `src/components/assistant/AssistantPanel.tsx:105`
**Issue:** Uses `parseMarkdown()` with custom regex-based HTML escaping instead of DOMPurify.
**Action:** Replace custom sanitizer with `DOMPurify.sanitize()` (already a project dependency).

### P2-4: In-Memory Cache Without Cleanup
**Files:**
- `server/routes/admin.ts:182` — Suggestions cache (`Map` with 1-hour TTL, no eviction)
- `server/lib/agent/claude/session-store.ts` — Thread/message persistence in memory

**Issue:** Memory grows unbounded in long-running processes.
**Action:** Add periodic cache eviction or move to Redis (already available in the stack).

### P2-5: Uncleaned setInterval in Auth
**File:** `server/lib/auth.ts:203`
**Issue:** `setInterval()` created without visible cleanup path.
**Action:** Store the interval ID and expose a cleanup function for graceful shutdown.
